01 — Overview

What is OpenClaw?

```

OpenClaw is a self-hosted AI gateway that bridges your favorite chat apps — WhatsApp, Telegram, Discord, iMessage, and more — to AI coding agents and LLMs. You run a single Gateway process on your own machine (or a server), and it becomes the bridge between your messaging apps and an always-available AI assistant.

"At this point I don't even know what to call OpenClaw. It is something new. After a few weeks in with it, this is the first time I have felt like I am living in the future since the launch of ChatGPT." — @davemorin
🏠
Self-Hosted
Runs on your hardware, your rules. Your data stays yours — private by default.
💬
Multi-Channel
One Gateway serves WhatsApp, Telegram, Discord, and more simultaneously.
🧠
Agent-Native
Built for coding agents with tool use, sessions, memory, and multi-agent routing.
📖
Open Source
MIT licensed, community-driven. Fork it, hack it, make it yours.

Formerly known as Clawdbot and Moltbot. Built by Peter Steinberger and the community. Independent project, not affiliated with Anthropic.

```

03 — Installation

Installing OpenClaw

```

System Requirements: Node 24 (recommended) or Node 22.16+. macOS, Linux, or Windows (WSL2 recommended).

⚠️
Ubuntu Version Matters! Use Ubuntu 24.04 LTS for AMD GPU setups. Ubuntu 25.x has kernel 6.14 which breaks ROCm DKMS builds.
1

Install via one-liner

curl -fsSL https://openclaw.ai/install.sh | bash
2

Fix PATH if needed

echo 'export PATH="$(npm prefix -g)/bin:$PATH"' >> ~/.bashrc
```

source ~/.bashrc
3

Run onboarding

openclaw onboard –install-daemon
4

Verify the Gateway

openclaw gateway status
```

Verify Your Install

openclaw --version      # confirm the CLI is available
```

openclaw doctor         # check for config issues
openclaw gateway status # verify the Gateway is running

03b — GPU Setup

AMD GPU + Ollama Setup

```

Run local models on AMD GPUs using ROCm and Ollama. This enables cost-free inference for sub-agents.

⚠️
Critical: Ubuntu 24.04 LTS Required Ubuntu 25.x ships with kernel 6.14 which breaks ROCm DKMS.

Step 1: Install ROCm 7.2

# Update system
```

sudo apt update && sudo apt upgrade -y

# Download ROCm installer

wget https://repo.radeon.com/amdgpu-install/7.2/ubuntu/noble/amdgpu-install_7.2.70200-1_all.deb

# Install the package

sudo apt install ./amdgpu-install_7.2.70200-1_all.deb

# Install ROCm and graphics drivers (takes 10-20 minutes)

sudo amdgpu-install -y –usecase=graphics,rocm

# Add user to required groups

sudo usermod -a -G render,video $LOGNAME

# REBOOT REQUIRED

sudo reboot
```

Step 2: Verify ROCm

rocminfo | grep -A 5 "Agent 2"
```

rocm-smi
```

Step 3: Install Ollama

curl -fsSL https://ollama.com/install.sh | sh

Look for "AMD GPU ready" at the end.

Step 4: Pull Models

ollama pull qwen2.5:14b    # Research (~9GB)
```

ollama pull llama3.1:8b    # Writing (~4.9GB)
ollama pull mistral:7b     # Creative (~4.4GB)
ollama list
```

Step 5: Verify GPU Acceleration

# Terminal 1: Run a query
```

ollama run qwen2.5:14b “Explain quantum computing in 50 words”

# Terminal 2: Check GPU usage

rocm-smi

07b — Discord Setup

Discord Bot Setup

```

Connect your OpenClaw agent to Discord. Your bot is private by default.

ℹ️
One Bot Recommended Set up one Discord bot (e.g., "Boss") that handles all interactions. Sub-agents don't need their own bots.

Step 1: Create Discord Application

1

Go to Discord Developer Portal

discord.com/developers/applications

2

Create Application

Click "New Application" → Name it (e.g., "OpenClaw Bot") → Accept ToS → "Create"

3

Configure Bot

Click "Bot" in sidebar → Set Username (e.g., "Boss")

Step 2: Get Bot Token

On the Bot page: "Reset Token" → Copy and save securely.

Step 3: Enable Privileged Intents

On Bot page, enable all three:

  • ✅ Presence Intent
  • ✅ Server Members Intent
  • ✅ Message Content Intent (required)

Click "Save Changes"

Step 4: Generate Invite Link

OAuth2 → URL Generator

Scopes: bot, applications.commands

Permissions: View Channels, Send Messages, Read Message History, Embed Links, Attach Files, Add Reactions

Copy the generated URL.

Step 5: Create Server & Add Bot

In Discord: Click "+" → "Create My Own" → "For me and my friends" (creates private server)

Paste invite URL in browser → Select server → "Authorize"

Step 6: Get Discord IDs

Enable Developer Mode: User Settings → Advanced → Developer Mode

Right-click server icon → "Copy Server ID"

Click your username → "Copy User ID"

Step 7: Configure OpenClaw

⚠️
Critical: Token Must Be in BOTH Places The token must be in your shell environment AND the systemd service file. Missing either causes gateway failures.

A. Set Token in Shell

# Add to bashrc
```

echo ‘export DISCORD_BOT_TOKEN=“YOUR_BOT_TOKEN”’ >> ~/.bashrc
source ~/.bashrc
```

B. Configure OpenClaw

# Configure token reference
```

openclaw config set channels.discord.token –ref-provider default –ref-source env –ref-id DISCORD_BOT_TOKEN

# Enable Discord

openclaw config set channels.discord.enabled true –strict-json
```

C. Add Token to systemd Service (CRITICAL)

# Create override directory
```

mkdir -p ~/.config/systemd/user/openclaw-gateway.service.d

# Create override file with token

echo ‘[Service]
Environment=“DISCORD_BOT_TOKEN=YOUR_BOT_TOKEN”’ > ~/.config/systemd/user/openclaw-gateway.service.d/override.conf

# Reload and restart

systemctl –user daemon-reload
systemctl –user restart openclaw-gateway

# Verify

openclaw gateway status
```

Step 8: Pair Your Account

DM the bot: hello

Bot responds with pairing code.

openclaw pairing approve discord <CODE>

Step 9: Enable Server Channels (Optional)

# Replace with your IDs
```

openclaw config set channels.discord.guilds ‘{“YOUR_SERVER_ID”: {“requireMention”: false, “users”: [“YOUR_USER_ID”]}}’ –strict-json

systemctl –user restart openclaw-gateway
```

Troubleshooting Discord

ProblemSolution
Gateway crashes on startupCheck journalctl --user -u openclaw-gateway -n 50
Token missing errorVerify override.conf has [Service] header
Bot doesn't respond in serverConfigure channels.discord.guilds
SIGTERM on startupStop conflicting service: systemctl --user stop openclaw-gateway
```

09 — Multi-Agent

Multi-Agent Setup

```

Run multiple agents with a main orchestrator delegating to specialized sub-agents.

ℹ️
Example Team Boss (cloud model) coordinates: Scout (research), Scribe (writing), Pixel (creative), Pulse (SEO) — all on local Ollama.

Step 1: Add Sub-Agents via CLI

openclaw agents add scout
```

openclaw agents add scribe
openclaw agents add pixel
openclaw agents add pulse

openclaw agents list
```

When prompted:

  • Workspace: ~/.openclaw/workspace/shared
  • Copy auth from main: No
  • Configure model: Yes → Ollama → Local mode

Step 2: Create models.json for Each Sub-Agent

# ~/.openclaw/agents/scout/agent/models.json
```

{
“providers”: {
“ollama”: {
“baseUrl”: “http://127.0.0.1:11434”,
“api”: “ollama”,
“models”: [
{
“id”: “qwen2.5:14b”,
“name”: “qwen2.5:14b”,
“reasoning”: false,
“input”: [“text”],
“cost”: {“input”: 0, “output”: 0, “cacheRead”: 0, “cacheWrite”: 0},
“contextWindow”: 32768,
“maxTokens”: 8192
}
]
}
},
“default”: “ollama/qwen2.5:14b”
}
```

Step 3: Enable Sub-Agent Permissions (CRITICAL)

openclaw config set agents.list.0.subagents.allowAgents '["scout","scribe","pixel","pulse"]' --strict-json
```

openclaw gateway restart
```
⚠️
Required! Without allowAgents, main agent can't spawn sub-agents.

Step 4: Create System Prompts

# ~/.openclaw/agents/scout/agent/SYSTEM.md
```

# Scout - Trend Researcher

You are Scout, the Trend Researcher.

## Your Job

- Research trending topics
- Analyze competitor content
- Compile research briefs

## Rules

- Stay factual and data-driven
- Only respond when Boss delegates to you

Step 5: Configure Team Info in USER.md

# ~/.openclaw/workspace/shared/USER.md

## The Team

You (Boss) lead a team of 4 AI sub-agents:

|Agent |ID    |Role      |Model             |
|------|------|----------|------------------|
|Scout |scout |Researcher|ollama/qwen2.5:14b|
|Scribe|scribe|Writer    |ollama/llama3.1:8b|
|Pixel |pixel |Creative  |ollama/mistral:7b |
|Pulse |pulse |SEO       |ollama/llama3.1:8b|

## How to Delegate

Use `sessions_spawn` tool to assign tasks.
```

Step 6: Test

rm -rf ~/.openclaw/agents/main/sessions/*
```

openclaw gateway restart
openclaw tui

Test: Use agents_list to show which sub-agents you can spawn

```

20 — Lessons

Battle-Tested Lessons

```

Ubuntu & ROCm

  • Ubuntu 24.04 LTS required — 25.x kernel breaks ROCm DKMS
  • Install ROCm before Ollama — Ollama detects GPU at install time

Multi-Agent

  • Add agents via CLI — Use openclaw agents add
  • Each sub-agent needs models.json
  • Team info in USER.md — not SYSTEM.md
  • Clear sessions after config changes

Discord + systemd

  • Token in TWO places — ~/.bashrc AND systemd override.conf
  • override.conf format strict — Must have [Service] header
  • SIGTERM on startup — Usually conflicting systemd service

Config Schema

  • agents.list[] array — Not agents.boss, agents.scout
  • allowAgents required — agents.list.0.subagents.allowAgents
```
